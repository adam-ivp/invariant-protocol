Why this exists

The invariant library is a living catalogue of DeFi failure modes. Every entry is grounded in a real bug — found, documented, and encoded as a formally verifiable constraint. Three patterns appear repeatedly across every protocol category.

Pattern 01
Lending Protocols
Governance asymmetry
Two functions, same intent, divergent side effects

The most reliable finding surface in DeFi: two governance or accounting functions that should be equivalent but aren't. One updates a critical bitmap, mapping, or accumulator. The other doesn't. The discrepancy is invisible at the call site. It surfaces in edge cases — eMode users, isolated positions, frozen assets — where the missing update silently bypasses the intended restriction. Governance acts to reduce risk. The protocol continues as if it didn't.

// Function A — updates enforcement bitmap ✓ function restrictAsset(address asset) currentConfig.setLtv(0); _updateEnforcementBitmap(asset); // propagates to all access paths // Function B — same stated intent, missing propagation ✗ function zeroAssetLtv(address asset) currentConfig.setLtv(0); // bitmap never updated // edge case users unaffected — restriction silently bypassed
Active findings in this category · IVP invariant: emode_ltv_consistency
Pattern 02
Stablecoin CDPs
Incomplete fix propagation
Patch applied to one mirrored path. Not the other.

When a codebase has structurally mirrored execution paths — free/paid, stable/variable, normal/emergency — a fix to one path leaves an identical vulnerability in its mirror. The ghost share pattern: after a full repayment, rounding leaves total debt at zero while total shares stays non-zero. The next borrower enters the wrong branch, gets undervalued shares, and existing shareholders extract value at their expense. Triggers through normal usage. No attack required.

// Path A — FIXED if (totalSharesA == 0 || totalDebtA == 0) { shares = debt; // 1:1 on fresh start } // Path B — NOT FIXED (structural mirror) ✗ if (totalSharesB == 0) { // missing: || totalDebtB == 0 shares = debt; } else { shares = mulDivDown(debt, totalSharesB, totalDebtB); // totalDebtB == 0, totalSharesB > 0 → ghost share extraction }
Active findings in this category · IVP invariant: debt_share_accounting
Pattern 03
Bridge Protocols
Event integrity divergence
Contract computes the right value. Event emits the wrong one.

Bridge relayers on destination chains use emitted events as the source of truth for settlement. When a swap contract correctly computes received tokens via balance delta but emits the user-supplied input amount instead, fee-on-transfer tokens produce a systematic divergence. The relayer sees an overstated amount and pays out accordingly. The difference comes from the bridge. PoC verified on mainnet fork.

// Correct: balance delta captures actual received uint256 received = token.balanceOf(address(this)) - balanceBefore; // Wrong: event emits user-supplied amount, not received ✗ emit Swap(fromToken, toToken, fromAmount, ...); // ^^^^^^^^^^ user-supplied // for fee-on-transfer tokens: fromAmount > received // relayer overpays — difference sourced from bridge liquidity

Mechanism
01
Specify

Protocol teams write invariants in ISL — a typed formal constraint language. Each spec is committed on-chain as a hash. The scope of coverage is exactly what's signed. Nothing implied.

02
Prove

An assigned prover fetches live state, Merkle-commits the snapshot, and runs the SP1 ZK circuit against all active invariants. Commit-reveal prevents front-running. Batch proofs cover N epochs atomically.

03
Verify

256-block finality window. Bonded dispute mechanism — disputer posts IVP, loses bond on failed dispute. SpotChecker fires on random epochs: a second prover re-proves independently, without knowing they're being checked.

04
Settle

Violation proven and finalized. Claim filed. Junior tranche absorbs first, senior if needed. Pro-rata distribution. No vote. No committee. The math settles it.

Design

ISL is the interface. Everything else is downstream.

ISL — the center of gravity

The ZK circuit, the prover network, and the coverage vaults all enforce what gets written in ISL. If auditors encode exploit classes in ISL, if researchers publish shared specs, if protocols fork the library — ISL becomes a standards layer. That's where protocol moats form. A shared invariant spec for lending solvency, written once and adopted across a category, is more valuable than any individual contract.

Semantic honesty

The hardest problem in invariant systems is not the ZK. It's proving the invariant was the correct invariant. IVP does not solve this — no system can. Coverage means specified properties held. The protocol team owns spec completeness. IVP enforces what is committed, completely and continuously.

Adversary-designed

Liveness recovery prevents a silent prover from halting the system. Bonded disputes prevent griefing. Proof size caps bound verifier gas. Anti-Sybil cooldowns. Dispute limits. The SpotChecker's rotating private seed makes check epochs unpredictable — hidden checker identity, commit-reveal, collusion cost amplification.

The unsolved economic problem

Slash amounts are finite. Exploit concealment value is not. For large protocols, off-chain bribery can economically dominate any slash. The current model — tiered slashing, SpotChecker, anomaly tracking — is the defense at this stage. The full solution requires watchdog markets, public replay bounties, and forced proof redundancy. This is the active research direction. We say this out loud.

Open problems — named honestly

Slash vs bribery ceiling. Finite slash amounts cannot outcompete infinite exploit value at scale. Watchdog markets and replay bounties are the direction.

Prover collusion is operationally mitigated by SpotChecker. Not cryptographically eliminated.

Spec completeness is a protocol governance decision. Under-specified = under-covered. Nothing implied.

Audits ask: was the code good
at deployment?

IVP asks: are the security
assumptions still true right now?

That's an enormous conceptual difference. An audit is a snapshot. The protocol ships, upgrades, adds collateral types, integrates new bridges, composes with other protocols. The snapshot doesn't follow it.

IVP does. Not by monitoring — by verifying. Every epoch, a ZK prover checks whether the specified properties still hold against live state. A property breaks, the proof is on-chain, Guardian fires. The attacker's drain gets intercepted before completion.

What protocols actually buy: five critical invariants and freeze capability for catastrophic state divergence. Accounting conservation. Solvency. Collateralization. Bridge escrow integrity. Share/debt parity. Real pain points. Real exploit classes.
